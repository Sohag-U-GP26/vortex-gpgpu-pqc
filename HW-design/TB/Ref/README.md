# Ref — Golden Reference Vectors (Phase 1)

> Part of [Phase 1: Standalone RTL Architecture](../../README.md) — golden reference vectors for RTL cross-validation.

## Purpose

This directory contains the golden reference vectors used by the SystemVerilog testbenches to verify the correctness of the RTL implementation. These vectors are generated by the Python golden model (`Golden-python-model/kyber_ntt.py`) and serve as the ground truth for cross-validation.

## How Reference Vectors Are Created

The Python golden model produces all reference vectors in a single execution:

```bash
cd Golden-python-model/
python kyber_ntt.py
```

The script:
1. Generates two random 256-coefficient polynomials (A and B) using seed 42
2. Computes NTT(A), NTT(B), basecase product, INTT, and final result
3. Exports all intermediate and final results to `TB/Ref/`

## File Overview

### NTT Test Vectors

| File | Format | Description |
|------|--------|-------------|
| `a_ntt_input.txt` | Decimal, 1 per line | 256 coefficients of polynomial A |
| `a_ntt_output.txt` | Decimal, 1 per line | NTT of polynomial A (128 pairs) |
| `a_ntt_output.hex` | Hexadecimal | NTT(A) in hex (used by basemul TB) |
| `b_ntt_input.txt` | Decimal, 1 per line | 256 coefficients of polynomial B |
| `b_ntt_output.txt` | Decimal, 1 per line | NTT of polynomial B (128 pairs) |
| `b_ntt_output.hex` | Hexadecimal | NTT(B) in hex (used by basemul TB) |

### Basecase Multiplication Vectors

| File | Format | Description |
|------|--------|-------------|
| `basemul_output.txt` | Decimal, 1 per line | Basecase multiplication of NTT(A) × NTT(B) — 256 coefficients |
| `basemul_output.hex` | Hexadecimal | Same data in hex format |

### INTT and Final Result Vectors

| File | Format | Description |
|------|--------|-------------|
| `intt_output.txt` | Decimal, 1 per line | INTT of basecase product — **before** final n⁻¹ scaling |
| `final_output.txt` | Decimal, 1 per line | Final result after ×3303 mod 3329 — the expected output of the complete pipeline |

## How Testbenches Use Reference Vectors

### NTT Testbench (`tb_ntt_full.sv`)

```systemverilog
// Load input
fd = $fopen("Ref/a_ntt_input.txt", "r");
for (int i = 0; i < N; i++)
    void'($fscanf(fd, "%d", tmp));
    poly[i] = 12'(tmp);

// Load expected output
fd = $fopen("Ref/a_ntt_output.txt", "r");
for (int i = 0; i < N; i++)
    void'($fscanf(fd, "%d", tmp));
    ref_out[i] = tmp;

// Compare after simulation
if (poly[i] == ref_out[i]) → PASS
else → FAIL
```

### Basecase Testbench (`tb_basemul_kyber_v2.sv`)

```systemverilog
$readmemh("a_ntt_output.hex", A_mem);
$readmemh("b_ntt_output.hex", B_mem);
$readmemh("basemul_output.hex", C_exp);
```

### INTT Testbench (`tb_intt_full.sv`)

```systemverilog
// Load input (basecase output) — note: uses ../Ref/ because TB runs from TB/
fd = $fopen("../Ref/basemul_output.txt", "r");

// Load expected output (final product)
fd = $fopen("../Ref/final_output.txt", "r");
```

## Data Format

All `.txt` files contain one integer per line in decimal format. The values are 12-bit unsigned integers in the range [0, 3328] (i.e., modulo q = 3329).

All `.hex` files contain one hexadecimal value per line, packed MSB-first, with values in the same range.

## Regenerating Vectors

The Python golden model uses `random.seed(42)` for reproducibility. As long as the seed and parameters remain unchanged, regenerating the vectors produces identical output.

To regenerate:

```bash
python Golden-python-model/kyber_ntt.py
```

After regeneration, all RTL testbenches should automatically pick up the new vectors and produce consistent results. If a testbench fails after regeneration, either:
- The Python model was modified (algorithm change)
- The RTL was modified (and needs updating)

## Adding New Reference Vectors

To support testing of a new module or pipeline stage:

1. In the Python golden model, compute the new intermediate or final result
2. Export to a new file using `json.dump()` or manual file writing
3. Place the file in this directory with a descriptive name
4. Update the relevant testbench to read from the new file
5. Document the new file in this README
